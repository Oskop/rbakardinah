# Walkthrough - Phase 5: Budget Logic & Locking

I have implemented the budget ceiling (Pagu) management for Administrators and enforced budget locking for Operators.

## Key Accomplishments

### 1. Admin Pagu Management
- Created `RbaAccountPaguController` for the Administrator role.
- Implemented a "Set Pagu Global" interface where Admin can see total requests per account code and set a nominal limit.
- Added a button in the RBA Header detail page to access the Pagu management interface.

### 2. Security & Locking (Policies)
- Reinforced `RbaDetailPolicy` to check the `rba_account_pagus` table.
- **Locking Logic**: If an Administrator has set a Pagu for a specific account code in an RBA Header, Operators can no longer add new items or update nominal amounts for that account code in their unit submissions.

### 3. Operator Pagu Indicators
- Updated the Operator Workboard to display:
  - **Pagu Global**: The limit set by Admin for each account.
  - **Status Pagu**: Visual indicators (✓ Tercover, MELEBIHI PAGU, or Pagu Belum Diset).
  - **Over Budget Warning**: Displays the excess amount if the total unit requests exceed the global pagu.
